# Documentation

Project-specific documentation lives here. Add architecture notes, deployment runbooks, and feature specs as they're written.

## Suggested structure

- `architecture.md` — high-level system overview (frontend, backend, database)
- `deployment.md` — how to deploy and configure environments
- `database.md` — schema notes, migrations, seed data
- `branding.md` — current company identity, color tokens, logo usage

> Older project reference material (design HTML mocks, business requirement docs) is kept locally in `_archive/old-project-refs/` (gitignored) for inspiration only.
